# EvoLab 最终目标与提交审查

审查依据是 DGX Spark Hackathon 的六项评审标准和六项提交要求。结论分成两部分：仓库内交付已经具备提交条件；团队合影、主办方提交结果和外部征文链接仍要在仓库之外完成。

## 项目目标是否落地

EvoLab 的目标不是随机生成一张陌生生物图，而是让用户在七个世界中连续改变环境、偶发事件和演化方向，并看见谱系怎样继承、能力带来什么代价、哪些结论有证据，哪些只是有限推测。

当前主链已经覆盖这几个目标：

- 七个世界各有起点、三轮规则、知识卡、证据模式和视觉约束；
- 三轮会话保存父代、继承性状、受保护性状、用户选择和阶段图片；
- 历史路线先匹配真实类群及内外部性状，找不到完整对应时明确收窄为有限推测；
- Step 规划 Agent 产出严格结构，独立科学审查 Agent 可以放行、要求一次修订或阻断；
- DGX Spark 本地运行 ComfyUI、图片模型、会话服务和媒体任务；Klein 失败后回到保留父图条件的 FLUX.1；
- 结果图经过 Step 多模态连续性审查；能力不可用时只保留技术检查，不冒充语义判断；
- 三轮完成后生成四阶段回放，证据弹层同时说明来源和不能推出的结论。

## 提交前必须由团队完成

1. 上传真实团队合影。
2. 在主办方页面完成提交，并保存成功页面或回执。
3. 如果征文需要 CSDN、知乎或指定平台 URL，发布仓库中的“十日谈”，再把地址回填到 README 和提交页。
4. 在社交平台发布 AI 视频或图片时，按平台规则标注 AI 生成。

## 不应在答辩中越过的边界

- 本地 `--dry-run` 可以证明接口、流程和页面，不证明 DGX 当前模型已经加载；现场状态以 `/api/readiness` 为准。
- FLUX.1、FLUX.2 Klein 和 HunyuanVideo 不是 NVIDIA 模型。
- HunyuanVideo 短片证明设备侧 I2V 跑通，不是四阶段演化证据。
- 历史类群匹配是生成约束；部分匹配和无匹配不能被改写成发现了真实祖先。
